Food Delivery UX Transformation

Кейс по глубокому анализу и редизайну пользовательского пути в мобильном приложении доставки еды.

📌 Intro

Индивидуальный проект по поиску «болевых точек» (pain points) в текущем интерфейсе крупного сервиса доставки. 

Цель — сократить путь пользователя до оформления заказа и повысить конверсию в покупку.

🧪 UX Исследование

CustDev: Проведены глубинные интервью с 5 пользователями для выявления барьеров.

CJM & User Flow: Визуализирован текущий путь заказа; выявлено 3 критические точки отказа.

Анализ конкурентов: Проведен сравнительный аудит интерфейсов лидеров рынка (Яндекс.Еда, Delivery Club).

🔧 Решение (Prototype)

Инструментарий: Разработан интерактивный Hi-Fi прототип в Figma.

Оптимизация: Путь оформления заказа сокращен с 7 до 4 экранов.

Валидация: Проведено юзабилити-тестирование прототипа, по итогам которого внесены финальные правки в навигацию.

📈 Результаты

Устранение когнитивной нагрузки на этапе выбора адреса.

Готовый кейс, подтверждающий навыки работы с Figma, Miro и методологиями исследования пользователей.
